## Added vector max size allocation detection, parsing error detection for vtkHDRReader and better overall protection

Added simple vector max size allocation detection to the `vtkHDRReader` so that the std::vector lineBuffer won't attempt to create a bigger vector than the max_size() allowed. Additionally ensured that when attempting to read in the dims of the image a: if the parsing fails it will return an error and not use unallocated dimention variables, and b: the unallocated dimention variables are now defaulted to 0. Additionally when a ExecuteInformation() fails ExecuteDataWithInformation() will not be attempted with garbled data. This ensures that when reading a HDR file that is invalid (e.g. garbled header or cut-off file) the reader won't try to read data past the end of the file contents vector.
